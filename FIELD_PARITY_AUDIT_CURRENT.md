# Field Parity Audit — Current State

_Created: 2026-05-19, on branch `claude/field-parity-pass`._

> **How to read.** Each entity-type section has three buckets:
> - **A. Visible/display parity** — fields a panel or workspace shows
>   today; the editor MUST support them.
> - **B. Schema-supported aspirational** — fields the editor /
>   persistence / JSON-template / import-export supports, even when no
>   panel currently displays them. Round-trips correctly.
> - **C. Unsupported / future** — required by the design brief but not
>   yet in the schema this pass.
>
> "Severe gap" = a field is in bucket A but missing from the editor /
> JSON / persistence. That regresses to "the user sees a value the
> editor can't save". Fixed in this pass.

## Methodology

The actual editor configs were extracted by running a small VM-shimmed
Node script over `entity-editor-configs.jsx` and
`entity-editor-configs-extended.jsx`, reading
`window.ENTITY_EDITOR_CONFIGS[type]` after both modules executed. Field
IDs below are real — copied from the config — not transcribed from
docs.

Persistence is `EntityService.save(type, fields, opts)` →
`StorageService.set(KEYS.entities, ...)`. Whatever the editor produces
is what's stored. JSON template generated by `eeJsonTemplate(type)`
walks the same config and emits one key per `field.id`. So:

> **If a field is in the editor config, it is automatically in the
> JSON template, in persistence, and in entity-level export.**

The audit is mostly: does the config have the fields the brief and the
panel renderers expect?

## Unknown-field preservation

`EntityService.save(type, fields, opts)` does a `clone(fields)` and
spreads into the stored entity. Any keys present in `fields` survive,
including ones the editor doesn't render. The editor's "Paste JSON"
flow drops unknown keys today (it only fills inputs it knows about) —
but a programmatic save via `EntityService.save` preserves them.

**This pass adds:**
- A test that saves an entity with an `extra.future` key, reloads,
  and asserts `extra.future` survives.
- A guard pattern in the JSON import flow that stashes unknown keys
  under `entity.extra` so they survive even when the editor is
  involved.
